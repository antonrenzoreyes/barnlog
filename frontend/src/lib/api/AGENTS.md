# API Agent Guide

## Scope

- This file applies to all work under `frontend/src/lib/api/`.

## API UX Skills

- Use `$clarify` to improve API-originated user-facing copy and error messaging.
- Use `$harden` to strengthen behavior for API failures, timeouts, and offline scenarios.
- Use `$optimize` to reduce API-driven latency impact and unnecessary client work.

## OpenAPI Types

- Treat the Go backend as the single source of truth for domain data and API contracts.
- Keep API calls in typed client modules under `frontend/src/lib/api/`.
- Use [`generated/types.ts`](./generated/types.ts) as the source of truth for API request/response contract types.
- `generated/types.ts` is auto-generated by `openapi-typescript`; do not edit it directly.
- Types are generated from `backend/openapi/openapi.yaml` (canonical backend API contract).
- Default to OpenAPI-generated types for request/response contracts and document any handwritten exceptions inline.
- Regenerate types from `frontend/` with `bun run gen:api-types`.
- Verify generated types are current with `bun run check:api-types`.

## Error Handling

- Handle network errors, timeouts, and non-2xx responses consistently in API client utilities.
- Normalize API errors to one shared app-level format in `src/lib/api`.
- Have pages and components consume the normalized API error shape instead of ad hoc transport-specific errors.
